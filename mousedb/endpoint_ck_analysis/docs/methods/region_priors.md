# Region priors

All plots in this tool display brain regions in a specific canonical order
rather than alphabetical or data-driven order. The order encodes a
predicted-importance ranking for skilled forelimb reaching.

**Source of truth**: `mousedb.region_priors.SKILLED_REACHING` when the
live mousedb package is installed, or `endpoint_ck_analysis.config.FALLBACK_SKILLED_REACHING`
(a frozen snapshot) otherwise. Import via `endpoint_ck_analysis` which picks one.

---

## Why ordering matters

When a heatmap has regions on its y-axis, alphabetical ordering puts
"Cerebellospinal Nuclei" next to "Corticospinal" next to "Dorsal Reticular
Nucleus" for no reason. Predicted-importance ordering puts the regions
most likely to matter for reaching at the top and the "kept for contrast"
regions at the bottom, so the eye naturally compares within groups.

A dashed red line at the `high_priority_cutoff` index separates the two
groups.

## Customizing the ordering

Two cases:

- **One-off** (just for a local figure): use `ordered_hemisphere_columns`
  with your own list directly. Don't change `SKILLED_REACHING`.
- **New activity** (e.g. skilled locomotion): add a new `RegionPrior`
  constant in `mousedb/region_priors.py`, register it in `PRIORS`, and
  import from there. The fallback snapshot in `endpoint_ck_analysis/config.py`
  should be regenerated by `tools/sync_region_priors.py` when mousedb is
  re-exported.

## `ordered_hemisphere_columns` helper

All the PCA / PLS / heatmap columns look like `{region}_{hemisphere}`
(e.g. "Red Nucleus_both", "Red Nucleus_left", "Red Nucleus_right"). The
helper function builds these column names in prior order, optionally
filtered to what your DataFrame actually has:

```python
from endpoint_ck_analysis import SKILLED_REACHING, ordered_hemisphere_columns
ordered = ordered_hemisphere_columns(SKILLED_REACHING, available=df.columns.tolist())
df = df[ordered]   # Now in canonical order
```
