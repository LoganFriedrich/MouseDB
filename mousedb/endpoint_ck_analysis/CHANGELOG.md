# Changelog

Audience: "did the figures or analytical choices change?". One line per
release is the goal.

## 0.1.0 - 2026-04-23

Initial release. Refactor of the monolithic CFS_Project notebook into a
distributable subpackage of `mousedb`.

- Six Python helper modules under `endpoint_ck_analysis/helpers/` covering
  kinematic feature selection, connectivity pivoting, subject filtering,
  PCA/PLS dimensionality reduction, linear mixed models, and plotting.
- Nine notebooks: `00_setup`, `01_connectivity_pca`, `02_kinematic_pca`,
  `03_kinematic_clustering`, `04_pls_variants`, `05_lmm_phase_effects`,
  `06_pellet_validation`, `07_connectivity_trajectory_linkage`, and
  `99_figure_gallery`. Notebook 07 contains the full end-of-pipeline
  analysis: ML-driven clustering (ward/kmeans/gmm/consensus), cluster
  profiling with auto-naming, permutation-based cluster validation,
  alluvial/Sankey visualization of per-phase kinematic regrouping,
  trajectories colored continuously by connectivity PC1 and
  categorically by named cluster, and a nested-random-effects interaction
  LMM. Supports a ``USE_SYNTHETIC`` flag that swaps real data for a
  cloned-and-perturbed cohort generated by
  ``helpers.synthetic.synthesize_cohort`` so the pipeline can be
  validated at realistic N while real N is still small.
- Two additional helper modules:
  ``endpoint_ck_analysis/helpers/clusters.py`` (method-agnostic clustering,
  profiling, auto-naming, permutation validation, alluvial records)
  and ``endpoint_ck_analysis/helpers/synthetic.py`` (prototype-and-perturb
  cohort generator with encoded lineage in synthetic subject IDs).
- Notebook ``08_hypothesis_informed_tests`` and supporting
  ``endpoint_ck_analysis/helpers/hierarchical.py`` module for: grouped-vs-
  ungrouped connectivity PCA, per-eLife-group drill-down on atomic
  regions, nested LMM comparison (phase-only vs + top-k priority
  regions), prior-weighted PCA. ``dimreduce.priority_weights_from_prior``
  and ``dimreduce.apply_feature_weights`` added to support the
  priority-weighted decomposition.
- Memory note filed for paper-readiness methods deferred until N scales:
  tensor decomposition, cross-validated prediction, supervised
  clustering, session-level growth curves, outlier-aware modeling.
- Parquet cache for the six base dataframes so downstream notebooks never
  re-touch the SQLite database.
- Soft-imports `mousedb.region_priors` with a frozen fallback so the tool
  runs in environments without the parent package installed.
- Windows `.bat` and macOS/Linux `.command` launchers for non-technical
  users.
- `doctor()` health check exposed in `00_setup` to surface environment
  problems up front.

Analytical behavior matches the source notebook byte-for-byte: same
aggregations, same PCA/PLS configurations, same LMM model spec (nested
session-in-subject random intercepts) against the same `connectome.db`.
